# 상식 퀴즈 게임 — Claude Code 구현 프롬프트 (3단계)

---

## 1단계: 문제 데이터 생성

```
상식 퀴즈 게임에 사용할 문제 JSON 파일 12개를 생성해 줘.

## 파일 구조
questions/ 폴더 안에 아래 12개 파일을 만들어:
- korean_history_easy.json / korean_history_normal.json / korean_history_hard.json
- science_easy.json / science_normal.json / science_hard.json
- geography_easy.json / geography_normal.json / geography_hard.json
- general_easy.json / general_normal.json / general_hard.json

## 각 파일 형식
파일마다 10개의 문제 객체를 담은 JSON 배열. 각 문제는 아래 구조를 따라:

{
  "category": "한국사",           // 한국사 | 과학 | 지리 | 일반상식
  "difficulty": "easy",           // easy | normal | hard
  "id": "kor_easy_001",           // 카테고리+난이도+번호
  "question": "문제 텍스트",
  "options": ["보기1", "보기2", "보기3", "보기4"],
  "answer": 0,                    // 정답 보기의 인덱스 (0~3)
  "explanation": "정답 핵심 해설 1~2줄",
  "option_explanations": [
    "보기1에 대한 설명",
    "보기2에 대한 설명",
    "보기3에 대한 설명",
    "보기4에 대한 설명"
  ]
}

## 난이도 기준
- easy: 교과서 기본 지식, 널리 알려진 사실
- normal: 중학교~고등학교 수준, 약간의 배경지식 필요
- hard: 심화 지식, 세부 사항, 혼동하기 쉬운 개념

## 카테고리별 출제 범위
- 한국사: 고대~현대 역사, 주요 인물, 사건, 왕조
- 과학: 물리·화학·생물·지구과학 기초 개념 및 법칙
- 지리: 세계 국가, 수도, 지형, 기후, 대륙
- 일반상식: 문화, 사회, 스포츠, 언어, 시사

## 주의사항
- 모든 문제는 4지선다로, 보기 중 정답은 1개
- option_explanations는 정답 보기도 포함해 4개 전부 작성
- 문제와 보기는 한국어로 작성
- 동일한 문제가 easy/normal/hard에 중복되지 않도록 할 것
```

## 2단계: 핵심 로직 및 UI 구현

```
상식 퀴즈 게임의 HTML/CSS/JS 전체를 구현해 줘.
questions/ 폴더에 JSON 파일 12개가 이미 준비되어 있어.

## 파일 구성
- index.html       — 전체 화면 마크업 (SPA, 화면은 show/hide로 전환)
- style.css        — 전체 스타일
- js/main.js       — 앱 진입점, 화면 전환 제어
- js/quiz.js       — 문제 로드(fetch), 출제, 채점 로직
- js/timer.js      — 타이머 카운트다운, 프로그레스 바 제어
- js/score.js      — 점수·콤보·등급 계산
- js/leaderboard.js — LocalStorage 순위 저장/조회
- js/share.js      — 결과 공유 텍스트 생성 및 클립보드 복사

## 화면 목록 (총 6개, CSS display로 전환)
1. 시작 화면     — 닉네임 입력 + 카테고리 선택 버튼 4개 + 리더보드 버튼
2. 난이도 화면   — 쉬움/보통/어려움 버튼 (각 난이도 설명 포함)
3. 퀴즈 화면     — 상단 정보바 + 타이머 프로그레스 바 + 문제 + 보기 4개
4. 피드백 화면   — 정오답 표시 + 보기별 오답 분석 + 다음 문제 버튼
5. 결과 화면     — 최종 점수·등급·정답률·최고콤보 + 저장·공유·다시하기 버튼
6. 리더보드 화면 — 카테고리×난이도 필터 탭 + 상위 10명 테이블

## 게임 로직 상세

### 문제 출제
- fetch(`questions/${category}_${difficulty}.json`)으로 로드
- 로드한 배열을 Fisher-Yates 셔플 후 앞 10문제 사용
- 각 문제의 options 배열도 셔플 (answer 인덱스는 셔플 후 재계산)

### 타이머
- 난이도별 제한시간: easy=30초 / normal=20초 / hard=15초
- setInterval로 100ms마다 갱신, 프로그레스 바 width% 업데이트
- 남은 시간 5초 이하 시 바 색상을 #ef4444(빨강)으로 변경
- 시간 초과 시 자동 오답 처리 후 피드백 화면으로 이동

### 점수 계산 (score.js)
기본 점수: easy=5점 / normal=10점 / hard=15점

타이머 보너스 (남은 시간 비율 기준):
- 70% 이상 → +5점
- 40~69%   → +3점
- 10~39%   → +1점
- 10% 미만 또는 시간초과 → +0점

콤보 보너스 (연속 정답 수 기준):
- 3연속  → +5점 (달성 시 "3 COMBO!" 팝업)
- 5연속  → +10점 (달성 시 "5 COMBO!" 팝업)
- 10연속 → +20점 (달성 시 "10 COMBO!" 팝업)
- 오답 또는 시간초과 시 콤보 카운터 0으로 초기화

등급 (정답 수 / 10 기준):
- S: 90% 이상 / A: 75~89% / B: 60~74% / C: 40~59% / D: 40% 미만

### 피드백 화면
- 정답 보기: #22c55e 배경
- 선택한 오답 보기: #ef4444 배경
- 보기 4개 각각에 option_explanations 텍스트 표시
- 시간 초과 시 "시간 초과!" 메시지 추가
- "다음 문제" 버튼 클릭 시에만 진행 (자동 진행 없음)

### 리더보드 (leaderboard.js)
- LocalStorage 키: `quiz_scores_${category}_${difficulty}`
- 저장 데이터: { nickname, score, grade, correct, total, maxCombo, category, difficulty, date }
- 상위 10개만 유지 (score 내림차순 정렬 후 slice)
- 리더보드 화면에서 카테고리·난이도 탭 필터로 조회
- 현재 게임 결과 행은 배경색으로 하이라이트

### 공유 (share.js)
결과 화면의 "결과 공유" 버튼 클릭 시 아래 텍스트를 클립보드에 복사:
"""
📝 상식 퀴즈 결과
카테고리: {카테고리} | 난이도: {난이도}
점수: {점수}점 | 등급: {등급}
정답률: {correct}/10 ({정답률}%) | 최고 콤보: {maxCombo}연속
#상식퀴즈 #{카테고리}
"""
복사 성공 시 버튼 텍스트를 "복사됨!" 으로 2초간 변경

## UI/디자인 원칙
- 배경: #ffffff, 텍스트: #111827
- 정답 컬러: #22c55e / 오답 컬러: #ef4444 / 강조 컬러: #3b82f6
- 폰트: 'Noto Sans KR', sans-serif (Google Fonts CDN)
- 불필요한 애니메이션 없이 기능 중심 미니멀 레이아웃
- 반응형: max-width 600px 센터 정렬, 모바일 360px 이상 대응
- 키보드: 숫자 1~4로 보기 선택, Enter로 다음 문제 진행

## 주의사항
- index.html에서 모든 JS 파일을 <script type="module">로 로드
- fetch는 로컬 파일서버 환경 기준 (file:// 프로토콜 미지원 — 사용자가 로컬서버로 실행)
- LocalStorage 저장 실패 시 조용히 무시 (try/catch)
- 닉네임 미입력 시 게임 시작 불가, 경고 메시지 표시
```

## 3단계: 통합 검증 및 완성도 점검

```
상식 퀴즈 게임 구현이 완료됐어. 아래 항목을 순서대로 점검하고 발견된 문제를 모두 수정해 줘.

## 점검 항목

### 1. 데이터 무결성
- questions/ 폴더의 12개 JSON 파일이 모두 존재하는지 확인
- 각 파일에 문제가 정확히 10개 이상인지 확인
- 모든 문제의 answer 인덱스가 options 배열 범위(0~3) 내인지 확인
- option_explanations가 4개인지 확인
- JSON 파싱 오류가 없는지 확인 (JSON.parse 테스트)

### 2. 게임 플로우
- 시작 화면 → 카테고리 선택 → 난이도 선택 → 퀴즈 → 피드백 → 결과 → (저장) → 리더보드 전환이 모두 정상 작동하는지 확인
- 닉네임 미입력 시 시작 불가 처리가 되는지 확인
- "다시 하기" 버튼이 시작 화면으로 돌아가는지 확인
- 리더보드에서 "돌아가기"가 시작 화면으로 연결되는지 확인

### 3. 퀴즈 로직
- 문제와 보기가 매 게임 랜덤 셔플되는지 확인
- 셔플 후 answer 인덱스가 올바르게 재계산되는지 확인
- 타이머가 문제 표시와 동시에 시작되는지 확인
- 보기 선택 즉시 타이머가 정지되는지 확인
- 시간 초과 시 자동 오답 처리가 되는지 확인

### 4. 점수 계산
- 난이도별 기본 점수 (easy=5/normal=10/hard=15)가 맞는지 확인
- 타이머 보너스 구간 (70%/40%/10%)이 올바르게 적용되는지 확인
- 콤보 보너스가 3/5/10연속 기준으로 정확히 지급되는지 확인
- 오답 시 콤보 카운터가 0으로 초기화되는지 확인
- 등급이 정답률 기준으로 정확히 산정되는지 확인 (S/A/B/C/D)

### 5. 피드백 화면
- 정답 보기가 초록(#22c55e), 오답 선택 보기가 빨강(#ef4444)으로 표시되는지 확인
- option_explanations 4개가 각 보기 아래에 표시되는지 확인
- 시간 초과 시 "시간 초과!" 메시지가 표시되는지 확인
- "다음 문제" 버튼 없이 자동 진행되지 않는지 확인

### 6. 리더보드
- LocalStorage에 점수가 올바른 키(quiz_scores_{category}_{difficulty})로 저장되는지 확인
- 점수 내림차순 정렬 후 상위 10개만 유지되는지 확인
- 현재 게임 결과가 리더보드에서 하이라이트되는지 확인
- 카테고리×난이도 탭 필터가 정상 작동하는지 확인

### 7. 결과 공유
- "결과 공유" 버튼 클릭 시 지정된 형식의 텍스트가 클립보드에 복사되는지 확인
- 복사 후 버튼 텍스트가 "복사됨!"으로 2초간 변경되는지 확인

### 8. UI/반응형
- 모바일(360px) 기준으로 레이아웃이 깨지지 않는지 확인
- 키보드 1~4로 보기 선택, Enter로 다음 문제 진행이 작동하는지 확인
- 타이머 바가 5초 이하에서 빨간색으로 변경되는지 확인
- 콤보 달성 시 팝업 텍스트("3 COMBO!" 등)가 표시되는지 확인

## 수정 방식
- 발견된 문제는 즉시 해당 파일을 수정
- 수정 후 최종적으로 게임 전체 동작에 이상이 없음을 확인
- 점검 결과를 항목별로 요약해서 보고해 줘
```
