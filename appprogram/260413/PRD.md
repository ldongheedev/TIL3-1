# PRD: 냉장고 재료 인식 레시피 추천 웹앱

## 제품 개요

냉장고 사진 한 장으로 남은 재료를 자동 인식하고, 그 재료로 만들 수 있는 레시피를 추천하는 웹 애플리케이션.
핵심 가치는 **식재료 낭비 절감** — 유통기한 임박 재료부터 우선 활용할 수 있도록 돕는다.

### 기술 스택 기반

- 이미지 인식: `google/gemma-3-27b-it:free` (OpenRouter)
- 레시피 생성: `google/gemma-3-27b-it:free` 또는 `qwen/qwen3.6-plus` (OpenRouter)
- API 유틸리티: `src/openrouter.js` (기존 구현)

---

## 1단계 — MVP: 이미지 업로드 + 재료 인식

### 목표

사용자가 냉장고 사진을 올리면 재료 목록을 텍스트로 반환한다.
백엔드 없이 정적 HTML + Vanilla JS로 구현한다.

### 사용자 시나리오

1. 사용자가 웹 페이지에 접속한다.
2. 냉장고 사진을 드래그&드롭하거나 파일 선택으로 업로드한다.
3. "재료 인식" 버튼을 누른다.
4. AI가 이미지를 분석해 재료 목록(이름 + 수량/상태)을 화면에 표시한다.
5. 사용자가 목록을 직접 수정(추가/삭제)할 수 있다.

### 기능 요구사항

| 기능 | 설명 |
|------|------|
| 이미지 업로드 | 파일 선택 + 드래그&드롭, 미리보기 표시 |
| 재료 인식 | 이미지 → Base64 변환 후 `chatWithImage()` 호출 |
| 결과 표시 | 인식된 재료를 태그/리스트 형태로 렌더링 |
| 재료 편집 | 항목 추가 · 삭제 가능 |
| 로딩 상태 | 분석 중 스피너 + 버튼 비활성화 |
| 에러 처리 | API 실패 시 사용자 친화적 메시지 표시 |

### 기술 구현 사항

- 이미지를 `FileReader`로 Base64 인코딩하여 `image_url`에 `data:image/...;base64,...` 형식으로 전달
- OpenRouter API 키는 클라이언트 직접 호출 (1단계 한정, 2단계에서 서버로 이관)
- 단일 `index.html` 파일로 구성

### 완료 기준

- [ ] 사진 업로드 → 재료 목록 표시까지 30초 이내
- [ ] 일반 냉장고 사진에서 주요 재료 80% 이상 인식
- [ ] 모바일 브라우저에서 레이아웃 깨짐 없음

## 2단계 — 레시피 추천 + UI 고도화

### 목표

인식된 재료를 바탕으로 실제 만들 수 있는 레시피를 추천하고,
유통기한 임박 재료를 우선 사용하도록 제안한다.

### 사용자 시나리오

1. 1단계에서 재료 목록을 확인·수정한다.
2. 각 재료에 **유통기한** 또는 **임박 여부**를 선택적으로 입력한다.
3. "레시피 추천" 버튼을 누른다.
4. 레시피 카드 3~5개가 표시된다 (요리명, 소요시간, 난이도, 단계별 조리법).
5. 임박 재료를 최대한 활용하는 레시피가 상단에 표시된다.
6. 재료 필터(채식, 글루텐프리 등)를 적용할 수 있다.

### 기능 요구사항

| 기능 | 설명 |
|------|------|
| 레시피 생성 | 재료 목록 + 필터 조건을 프롬프트로 구성해 텍스트 모델 호출 |
| 임박 재료 우선 | 유통기한 임박 재료를 프롬프트에 강조 |
| 레시피 카드 UI | 요리명, 소요시간, 난이도, 재료 사용률, 조리 단계 표시 |
| 식단 필터 | 채식 / 비건 / 글루텐프리 / 유제품 제외 선택 |
| 재료 사용률 표시 | 보유 재료 중 몇 %를 사용하는지 시각화 |
| 반응형 UI | 카드 그리드, 모바일 대응 |

### 기술 구현 사항

- 레시피 응답 형식을 JSON으로 강제하는 시스템 프롬프트 설계
- `Node.js` Express 서버 도입 — API 키를 서버 사이드로 이관
- `fetch('/api/recognize')`, `fetch('/api/recipe')` 엔드포인트 분리
- 정적 파일은 Express에서 서빙

### 완료 기준

- [ ] 임박 재료를 포함한 레시피가 항상 1개 이상 상위 노출
- [ ] 레시피 응답이 일관된 JSON 구조로 파싱 성공률 95% 이상
- [ ] 필터 적용 시 조건에 맞지 않는 레시피 미포함

## 3단계 — 저장 · 즐겨찾기 · 히스토리

### 목표

레시피와 재료 인식 결과를 저장하고 재사용할 수 있도록 한다.
반복 사용자를 위한 개인화 경험을 제공한다.

### 사용자 시나리오

1. 마음에 드는 레시피를 "저장" 버튼으로 즐겨찾기에 추가한다.
2. 저장된 레시피를 별도 탭에서 모아볼 수 있다.
3. 이전에 분석한 냉장고 사진과 재료 목록 히스토리를 확인한다.
4. 자주 쓰는 재료("항상 있는 재료")를 고정 등록해 매번 인식 없이 자동 추가한다.
5. 레시피 카드에서 "이 레시피 다시 추천" 버튼으로 재생성한다.

### 기능 요구사항

| 기능 | 설명 |
|------|------|
| 레시피 저장 | 로컬스토리지 기반 즐겨찾기 저장·조회·삭제 |
| 분석 히스토리 | 날짜별 냉장고 스캔 이력 및 재료 목록 보관 |
| 고정 재료 등록 | 항상 보유한 재료(소금, 식용유 등)를 설정에 저장 |
| 레시피 재생성 | 저장된 재료 목록으로 새 레시피 재추천 |
| 공유 기능 | 레시피 카드를 이미지로 다운로드 또는 URL 공유 |
| 사용 통계 | 이번 달 절약한 재료 수 / 만든 레시피 수 표시 |

### 기술 구현 사항

- 저장소: `localStorage` (로그인 불필요, 브라우저 로컬 유지)
- 공유 URL: 레시피 내용을 Base64 인코딩해 쿼리스트링으로 전달
- 이미지 다운로드: `html2canvas` 라이브러리로 카드 캡처
- 히스토리 최대 보관: 최근 30건 (FIFO)

### 완료 기준

- [ ] 저장된 레시피가 브라우저 재시작 후에도 유지
- [ ] 히스토리에서 재료 목록 불러와 레시피 재생성 동작
- [ ] 공유 URL로 접근 시 해당 레시피 카드 자동 표시
