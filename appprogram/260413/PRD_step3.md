# PRD Step 3 — 사용자 인증 + 레시피 저장 + 대시보드

## 개요
회원가입/로그인 프로토타입 시스템을 구축하고, 사용자별 레시피 저장 및 개인 대시보드를 제공합니다.

## 기술 스택
- Step 1, 2 스택 전부 유지
- **상태 관리**: Streamlit session_state (프로토타입 DB)

## 핵심 기능

### 1. 회원가입 / 로그인 (프로토타입)
- [나의 레시피] 탭에 로그인 / 회원가입 탭 UI
- session_state로 사용자 정보 관리 (아이디, 비밀번호, 저장 레시피)
- 테스트 계정: demo / demo123
- 중복 아이디 검사, 비밀번호 확인

### 2. 레시피 저장
- [레시피 생성] 탭의 [💾 저장] 버튼 활성화 (로그인 시)
- 저장 시 저장 일시 기록
- 사용자별 독립적인 저장 공간

### 3. 개인 대시보드
로그인 후 표시:
- **저장된 레시피 수** (metric)
- **요리 횟수** (metric)
- **저장된 레시피 목록** (확장 가능한 카드)
  - 요리명, 저장 일시, 난이도, 조리시간, 칼로리, 조리 순서
  - [삭제] 버튼으로 개별 레시피 삭제

### 4. 사이드바 로그인 상태 표시
- 로그인 중: "✅ {username} 로그인 중"
- 비로그인: 로그인 안내 메시지

## 로컬 개발 환경 안내
- 현재 앱은 로컬 환경(내 컴퓨터)에서만 작동: http://localhost:8501
- session_state 기반이므로 서버 재시작 시 데이터 초기화 (프로토타입 한계)
- 실제 서비스는 별도 DB(SQLite, PostgreSQL 등) 연동 필요

## 완료 기준
- [ ] 회원가입/로그인 정상 동작
- [ ] 레시피 [저장] 버튼이 로그인 시 활성화
- [ ] [나의 레시피] 탭에서 저장된 레시피 확인
- [ ] 대시보드에서 통계(저장 수, 요리 횟수) 표시
- [ ] 저장된 레시피 삭제 기능 동작
