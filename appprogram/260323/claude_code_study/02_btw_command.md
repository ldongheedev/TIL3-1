# `/btw` 명령어 상세 분석

> **추가 시점**: 2026년 3월 (Claude Code v2.1.72+)

---

## 개요

`/btw` (By The Way)는 진행 중인 작업의 **대화 히스토리를 오염시키지 않고**
즉석 질문을 던질 수 있는 명령어다.

```
/btw 이 함수에서 반환하는 frame28 변수가 뭐였지?
/btw preprocess_canvas 에서 float32를 쓰는 이유가 뭐야?
```

---

## 동작 원리

```
일반 질문                          /btw 질문
─────────────────────────────     ─────────────────────────────
사용자: 이 변수가 뭐야?            사용자: /btw 이 변수가 뭐야?
Claude: ...                        Claude: (오버레이로 답변)
                ↓                                  ↓
    대화 히스토리에 남음              대화 히스토리에 남지 않음
    컨텍스트 토큰 소비                컨텍스트 토큰 절약
```

1. 현재 세션의 전체 컨텍스트를 공유하는 **임시 에이전트** 생성
2. 질문·답변이 **오버레이(팝업)**로 표시됨
3. 답변 후 오버레이 닫히면 메인 대화에 흔적 없이 사라짐
4. 부모 대화의 **프롬프트 캐시를 재사용**하므로 추가 비용 최소화

---

## 핵심 특징

| 항목 | 내용 |
|------|------|
| **지속성** | 비영구 — 대화 히스토리에 저장되지 않음 |
| **컨텍스트** | 현재 세션 전체 내용 접근 가능 |
| **타이밍** | Claude가 작업 중일 때도 사용 가능 |
| **비용** | 프롬프트 캐시 재사용 → 추가 비용 최소 |
| **토큰 절감** | 잦은 중간 질문 시 세션 토큰 최대 **50% 절감** |

---

## 제한사항

- **읽기 전용**: 파일 수정, 명령 실행, 검색 불가
- **단일 턴**: 후속 질문 불가 — 대화가 필요하면 일반 프롬프트 사용
- **신규 정보 없음**: 세션에 이미 있는 정보만 답변 가능

---

## 언제 써야 하는가?

| 상황 | 권장 도구 |
|------|-----------|
| 방금 Claude가 읽은 코드에 대한 빠른 질문 | `/btw` ✅ |
| 변수명·함수명 확인 | `/btw` ✅ |
| 코드 수정 작업 중 방향 확인 | `/btw` ✅ |
| 새로운 파일 탐색 필요 | 서브에이전트 |
| 여러 번 주고받아야 하는 논의 | 일반 프롬프트 |

---

## `/btw` vs 서브에이전트 비교

| 항목 | `/btw` | 서브에이전트 |
|------|--------|------------|
| 컨텍스트 | 부모 세션 전체 공유 | 빈 컨텍스트로 시작 |
| 도구 사용 | 없음 (읽기 전용) | 모든 도구 사용 가능 |
| 용도 | 알고 있는 것에 대한 질문 | 새로운 것 발견·실행 |
| 대화 흔적 | 남지 않음 | 남지 않음 |

---

## 실제 활용 예시 (이 프로젝트에서)

```
# Flask 라우트 작성 중 갑자기 궁금할 때
/btw preprocess_canvas가 반환하는 두 번째 값이 뭐였지?

# JS 코드 수정 중
/btw /predict API가 probs 외에 뭘 더 반환했더라?

# 모델 관련 작업 중
/btw mnist_mlp.pkl 에서 hidden_layer_sizes 검증 조건이 뭐야?
```

메인 대화에 이 질문들이 쌓이지 않으므로 컨텍스트가 깔끔하게 유지됨.
