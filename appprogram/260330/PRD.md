# PRD: 개인용 할 일 관리 앱 (Todo Manager)

## 1. 개요

### 제품 목표
매일 10~20개의 할 일을 효율적으로 관리하는 개인용 웹 앱. 브라우저에서 바로 실행되며, 새로고침 후에도 데이터가 유지된다.

### 기술 스택
- **언어:** 순수 HTML / CSS / JavaScript (프레임워크 없음)
- **저장소:** localStorage (서버 불필요)
- **실행 환경:** 모던 브라우저 (Chrome, Firefox, Edge, Safari)

## 2. 사용자 스토리

| # | 스토리 | 우선순위 |
|---|--------|--------|
| 1 | 할 일을 빠르게 추가하고 싶다 | 필수 |
| 2 | 할 일의 내용, 카테고리, 우선순위를 수정하고 싶다 | 필수 |
| 3 | 완료한 할 일을 체크해서 시각적으로 구분하고 싶다 | 필수 |
| 4 | 필요 없는 할 일을 삭제하고 싶다 | 필수 |
| 5 | 업무/개인/공부로 분류해서 보고 싶다 | 필수 |
| 6 | 오늘의 전체 진행률과 카테고리별 진행률을 한눈에 보고 싶다 | 필수 |
| 7 | 중요한 일을 우선순위로 표시하고 싶다 | 필수 |
| 8 | 새로고침해도 내 할 일 목록이 그대로 남아있어야 한다 | 필수 |

## 3. 기능 명세

### 3.1 할 일 추가
- 입력창에 할 일 내용을 입력 후 Enter 또는 [추가] 버튼으로 등록
- 등록 시 함께 설정하는 항목:
  - **카테고리:** 업무 / 개인 / 공부 (기본값: 업무)
  - **우선순위:** 높음 / 보통 / 낮음 (기본값: 보통)
- 빈 내용은 등록 불가 (유효성 검사)

### 3.2 할 일 목록 표시
- 전체 목록 기본 표시
- 상단 탭 또는 필터 버튼으로 카테고리별 필터링 (전체 / 업무 / 개인 / 공부)
- 각 항목에 표시되는 정보:
  - 완료 체크박스
  - 할 일 내용
  - 카테고리 뱃지
  - 우선순위 표시 (색상 또는 아이콘)
  - 수정 / 삭제 버튼
- 정렬 순서: 미완료 항목이 위, 완료 항목이 아래. 미완료 항목 내에서는 우선순위 높음 → 보통 → 낮음 순

### 3.3 완료 체크
- 체크박스 클릭 시 완료 상태로 전환
- 완료된 항목은 시각적으로 구분 (취소선 + 흐린 색상)
- 체크 해제로 미완료 상태로 되돌릴 수 있음

### 3.4 수정
- [수정] 버튼 클릭 시 인라인 편집 모드로 전환
- 내용, 카테고리, 우선순위 수정 가능
- [저장] 또는 Enter 키로 저장, [취소] 또는 Esc 키로 되돌리기

### 3.5 삭제
- [삭제] 버튼 클릭 시 확인 없이 즉시 삭제
- 완료된 항목 전체 삭제 버튼 제공 ("완료 항목 모두 삭제")

### 3.6 진행률 표시
화면 상단에 고정 표시되는 진행률 섹션:

```
오늘의 진행률  ████████░░  8 / 10 완료 (80%)

업무    ██████░░░░  3 / 5  (60%)
개인    ██████████  2 / 2  (100%)
공부    ████░░░░░░  3 / 7  (40% — 미완료 4개)
```

- 전체 진행률: 전체 완료 개수 / 전체 등록 개수
- 카테고리별 진행률: 해당 카테고리 완료 개수 / 등록 개수
- 등록된 항목이 없는 카테고리는 표시하지 않음

### 3.7 데이터 영속성
- 모든 CRUD 동작 즉시 localStorage에 저장
- 페이지 로드 시 localStorage에서 데이터 복원
- 저장 키: `todo_app_data`
- 저장 형식: JSON 배열

```javascript
  completed: false,           // 완료 여부

}
```

## 6. 파일 구조

```
260330/
├── index.html       # 앱 진입점, 마크업 구조
├── style.css        # 전체 스타일
└── app.js           # 앱 로직 (상태 관리, 렌더링, localStorage)
```

### app.js 내부 구성

```
State          - todos 배열 (단일 진실 공급원)
Storage        - load(), save() → localStorage 연동
CRUD           - addTodo(), updateTodo(), deleteTodo(), toggleTodo()
Filter         - 현재 선택된 카테고리 필터 상태
Render         - renderTodos(), renderProgress() → DOM 직접 조작
EventListeners - 추가/필터/완료/수정/삭제 이벤트 바인딩
Init           - DOMContentLoaded 시 데이터 로드 + 렌더링
```
